# 1. Introduction

## 1.1 Context and Motivation

Grammatical Error Correction (GEC) has emerged as a mature Natural Language Processing (NLP) task with substantial real-world applications in Computer-Assisted Language Learning (CALL), educational technology, and writing support systems (Dale and Kilgarriff, 2011; Ng et al., 2014). The field has experienced rapid growth over the past decade, driven by advances in neural machine translation, sequence-to-sequence architectures, and the release of large-scale error-annotated corpora such as the BEA-2019 shared task dataset (Bryant et al., 2019). Current systems have achieved reasonable performance on benchmark test sets, with F-scores exceeding 0.60 on some evaluation datasets (Kiyono et al., 2019).

However, despite these technical advances, GEC systems remain underutilized in real educational contexts. A critical gap exists between the computational sophistication of these systems and their pedagogical utility. Most deployed systems treat all grammatical errors identically, providing uniform correction regardless of error type or learner proficiency level. This one-size-fits-all approach ignores substantial evidence from second language acquisition (SLA) research showing that effective corrective feedback is context-dependent, cognitively calibrated to the learner's proficiency, and grounded in linguistic theory (Ellis, 2009; Bitchener and Storch, 2016).

The pedagogical limitations of existing systems are particularly pronounced for learners across different proficiency levels. A beginner-level learner (CEFR A1–A2) benefits from simple, explicit explanations targeting high-frequency error patterns, whereas an advanced learner (CEFR C1–C2) may require nuanced feedback addressing subtle stylistic issues. Furthermore, different correction approaches have distinct strengths: rule-based systems achieve high precision on commonly understood patterns but low recall; neural approaches offer broader coverage but lower confidence on specific error types. No single engine dominates uniformly across all error categories. This complementarity suggests that a principled hybrid approach, informed by error-type analysis, could substantially improve both correction quality and pedagogical effectiveness.

## 1.2 Problem Statement

Contemporary GEC systems face three interconnected limitations:

**First, error-type agnosticism:** Most systems lack error-type-specific mechanisms for deciding whether to apply a correction. They treat all corrections equally during inference, using global precision–recall thresholds rather than leveraging known strengths and weaknesses of individual engines on specific error categories. This ignores rich information from error taxonomies such as ERRANT (Bryant et al., 2017), which provides fine-grained error type labels (e.g., Article, Preposition, Subject-Verb Agreement). A hybrid system could exploit these labels to select corrections from the engine most reliable for each error type.

**Second, proficiency-blind feedback:** Feedback generated by existing systems is monolithic—identical explanations are provided to all users regardless of their CEFR proficiency level (Common European Framework of Reference for Languages). Research in SLA demonstrates that learners benefit from developmentally appropriate feedback (Mackey, Oliver and Leeman, 2003); advanced learners may perceive simple explanations as condescending, while beginners may find theoretically complex explanations demotivating. CEFR provides a structured framework (A1–B2, C1–C2) for calibrating feedback, yet no published GEC system systematically adapts feedback across proficiency bands.

**Third, insufficient empirical characterization of engine complementarity:** While hybrid approaches are intuitively appealing, the literature lacks detailed quantitative analysis of which error types each engine (rule-based vs. neural) handles well. Without this analysis, hybrid designs default to simple strategies (e.g., voting or averaging), missing opportunities to weight decisions by error-specific performance. A rigorous analysis of per-error-type precision using ERRANT would enable data-driven engine selection.

## 1.3 Research Questions

Three grammar correction systems (GECs) are used — a rule-based checker (LanguageTool), a general-purpose pre-trained language model (T5-small), and a model fine-tuned specifically for error correction (LoRA flan-t5-base) — to address two research questions:

**RQ1:** Does selecting each correction from whichever system is most accurate for that particular error type outperform any single system used alone?

**RQ2:** What types of errors does each correction approach handle best, and how can these differences be exploited?

## 1.4 Contributions

This dissertation makes four contributions:

1. **A hybrid edit-level selection mechanism** that selects each edit according to the empirical precision of its originating engine for its ERRANT error category, under a minimum-precision threshold and a non-overlap constraint (Section 4.3).

2. **Four CEFR-adaptive feedback templates** — from explicit direct correction at level A, through metalinguistic annotation at level B and indirect coded feedback at level C, to minimal flagging at level N — grounded in the SLA corrective-feedback literature reviewed in Section 2.5.

3. **A quantitative characterisation of engine complementarity**: empirical per-ERRANT-category precision priors showing that LanguageTool, T5-small, and LoRA flan-t5-base attain highest precision on distinct error categories, and that selecting edits by those priors yields a 248% improvement in F₀.₅ over the strongest individual engine on held-out data (Section 6).

4. **A publicly available Streamlit demonstrator and full reproduction pipeline** that makes every headline number in the dissertation independently verifiable.
